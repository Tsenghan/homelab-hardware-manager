# 硬件资产管理系统 - 需求文档 v2.1

## 项目定位
专为个人及 HomeLab 场景设计的硬件与虚拟化资产管理系统。支持多层次结构展示（硬件 → 存储池 → 系统 → 应用），旨在清晰管理物理资源与虚拟机的映射关系，并提供极佳的全局搜索与拓扑可视化体验。

## 技术栈
- **后端**：Python Flask + SQLite
- **前端**：Vue 3 + Vite + Element Plus (或类似组件库) + ECharts
- **项目路径**：`./`

## 四层数据模型
支持物理机与虚拟化环境的深度映射，层层关联：

### L1 - 硬件层 (Hardware)
**`computer`**
- `id` (主键)
- `name` (名称，如 "PVE-Server-1"、"NAS主节点")
- `ip` (带外/物理管理IP)
- `location` (物理位置，如 "机柜A-3U"、"书房桌面")
- `notes` (备注)

**`cpu`**
- `id`, `computer_id` (外键)
- `model` (型号)
- `cores` (核心数)
- `clock_speed_ghz` (主频)
- `purchase_date`, `price`, `notes`

**`ram`**
- `id`, `computer_id` (外键)
- `brand`, `model`
- `capacity_gb`
- `type` (DDR4/DDR5等)
- `slots_occupied` (已用槽位，如 "DIMM_A1")
- `purchase_date`, `price`, `notes`

**`disk`** (物理硬盘)
- `id`, `computer_id` (外键)
- `brand`, `model`
- `capacity_gb`
- `interface` (SATA/NVMe/USB等)
- `slot_info` (物理槽位，如 "背板-盘位1")
- `is_boot_disk` (是否物理系统盘)
- `purchase_date`, `price`, `notes`

### L2 - 存储与虚拟化层 (Storage & Virtual Disk)
**`storage_pool`** (存储池/逻辑卷)
- `id`, `computer_id` (外键)
- `name` (名称，如 "local-lvm", "data-zfs")
- `type` (ZFS / LVM / ext4目录 / Btrfs等)
- `disks` (JSON或关联表，映射到底层的 `disk_id`)

**`virtual_disk`** (虚拟磁盘/挂载点)
- `id`, `storage_pool_id` (外键)
- `name` (名称，如 "vm-101-disk-0.qcow2", "subvol-105-disk-0")
- `format` (qcow2 / raw / lxc-fs / 直通物理分区)
- `size_gb` (分配容量)
- `os_instance_id` (外键，挂载给哪个系统/虚拟机)

### L3 - 系统层 (OS / Hypervisor)
支持系统嵌套（如 PVE 宿主机与内部 VM/LXC 的关系）。
**`os_instance`**
- `id`, `computer_id` (外键，物理宿主机)
- `parent_os_id` (自引用外键，为空代表物理宿主机，有值代表其为 VM/LXC)
- `name` (实例名，如 "Proxmox VE 8", "OpenWrt-主路由", "Debian-Docker")
- `type` (PVE / ESXi / Windows / Linux VM / LXC 等)
- `ip_address` (该系统的主业务或管理 IP)
- `mac_address`
- `pcie_passthrough` (文本/JSON，记录直通的显卡或网卡)
- `notes`

### L4 - 应用层 (Application)
**`service`**
- `id`, `os_instance_id` (外键)
- `name` (服务名，如 "HomeAssistant", "Nextcloud", "OpenClash")
- `port` (端口号)
- `url` (访问地址)
- `protocol` (http/https/ssh/tcp等)
- `notes`

## 核心交互与 UI 需求

### 1. 全局搜索 (Spotlight 模式)
- 顶部常驻全局搜索框。
- 跨层级模糊匹配：输入服务名、IP、MAC地址、硬件型号或甚至存储池名称，下拉展示分类结果。
- **快速跳转**：点击搜索结果，主视图自动展开对应层级，并在右侧呼出详情抽屉。

### 2. 抽屉式交互 (Drawer) 与可视化
- **摒弃多级页面跳转**：左侧/中侧为主拓扑树或列表，点击任何节点（如某块物理硬盘、某个服务），右侧滑出 Drawer 面板进行查看与编辑，保持上下文不丢失。
- **存储溯源图 (Storage Trace)**：在 L3 虚拟机详情中，清晰展示 `该VM → qcow2文件 → 存储池(local-lvm) → 物理硬盘(NVMe) → 物理机` 的完整链路。

### 3. 数据录入优化
- 提供**分步向导 (Wizard)** 录入全新物理机：`1.基础信息 → 2.硬件(CPU/内存/硬盘) → 3.存储池规划 → 4.初始宿主系统`，降低心智负担。
- 提供硬件记录的**“克隆/复制”**功能，方便批量录入同型号内存和硬盘。

### 4. 仪表盘与统计 (ECharts)
- **资源分配可视化**：在物理机或 PVE 系统详情中，展示内存和存储的“超售/闲置”状态（如水波图、进度条）。
- 全局统计：物理机数量、硬盘总数与总容量、VM/LXC 存活数量、运行服务数量等。

## 数据库与数据流
- 采用 **SQLite**，文件路径：`data/hardware.db`。
- **导出/备份**：支持一键将全库导出为结构化 JSON 文件，便于数据迁移与备份。
- **初始化示例**：系统首次启动时，自动写入包含 PVE 嵌套、ZFS 存储池、qcow2 虚拟磁盘映射的完整、真实的示例数据。

## 输出要求
- **后端**：遵循 RESTful 风格的 API 接口，实现完整的 CRUD 功能及特定逻辑（如存储溯源查询）。
- **前端**：Vue 3 单页面应用（SPA），组件化开发，交互流畅，图表渲染正常。
